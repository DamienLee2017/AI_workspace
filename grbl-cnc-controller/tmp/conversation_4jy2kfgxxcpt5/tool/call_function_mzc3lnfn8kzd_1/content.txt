## Project Overview

This project provides two versions of GRBL-style CNC controllers:

| Version | Platform | Features | Use Cases |
|---------|----------|----------|-----------|
| **Arduino Version** | Arduino Uno/Nano/Mega | Serial communication, stable and reliable | Entry-level CNC, desktop engravers |
| **ESP32-S3 Version** | ESP32-S3 | WiFi network control, higher performance | Advanced applications, wireless control |

### Core Features

- ✅ G-code parsing and execution
- ✅ Three-axis stepper motor control (X/Y/Z)
- ✅ Serial communication (Arduino) / WiFi + Serial (ESP32-S3)
- ✅ Real-time position display
- ✅ Speed and feed rate control
- ✅ Home position setting and zero return
- ✅ Emergency stop function

## G-code Support

This controller supports standard G-code instruction set:

### Motion Commands

| G-code | Function | Example |
|--------|----------|---------|
| G0 | Rapid move | `G0 X10 Y20` |
| G1 | Linear feed | `G1 X50 Y30 F100` |
| G28 | Return to home | `G28` |
| G90 | Absolute coordinate | `G90` |
| G91 | Relative coordinate | `G91` |

### Setting Commands

| G-code | Function | Example |
|--------|----------|---------|
| M3 | Spindle clockwise | `M3 S1000` |
| M5 | Spindle stop | `M5` |
| F | Set feed rate | `F200` |
| S | Set speed | `S5000` |

## FAQ

### Q: Stepper motor jitters or loses steps?

A: Check the following settings:
- Driver current setting (refer to driver module manual)
- Power supply voltage is sufficient
- Step pulse frequency is not too high
- Motor wiring sequence is correct

### Q: ESP32-S3 cannot connect to WiFi?

A: Confirm:
- WiFi password is correct
- Router frequency band (ESP32-S3 supports 2.4GHz)
- Signal strength

### Q: G-code not recognized?

A: Check:
- G-code format is correct
- Unsupported commands are not used
- Coordinate parameters are within travel range

### 步进电机连接

#### A4988 驱动模块设置

1. **调节参考电压 (Vref)**

   ```
   Vref = Imax × 8 × Rs
   
   其中:
   - Imax = 步进电机额定电流 (A)
   - Rs = 传感电阻值 (通常为 0.05Ω 或 0.1Ω)
   
   示例:
   - 使用 1.5A 电机, Rs = 0.1Ω
   - Vref = 1.5 × 8 × 0.1 = 1.2V
   ```

2. **MS1/MS2/MS3 细分设置**

   | MS1 | MS2 | MS3 | 细分 | 步数/圈 (200步电机) |
   |-----|-----|-----|------|---------------------|
   | L   | L   | L   | 全步 | 200                 |
   | H   | L   | L   | 半步 | 400                 |
   | L   | H   | L   | 1/4  | 800                 |
   | H   | H   | L   | 1/8  | 1600                |
   | H   | H   | H   | 1/16 | 3200                |

3. **驱动模块安装方向**

   ```
   注意: 确保驱动模块方向正确
   - EN 引脚朝外
   - 散热片朝上
   - 模块底部不要接触金属外壳
   ```

#### 步进电机接线

标准 4 线 NEMA 17 步进电机接线：

```
电机线颜色代码 (常见):

┌──────────────────────────────────────────────────┐
│  双极性步进电机                                   │
│                                                   │
│  A+  ── 蓝/绿 (绕组1)                             │
│  A-  ── 红/灰 (绕组1)                             │
│  B+  ── 橙/黄 (绕组2)                             │
│  B-  ── 黄/白 (绕组2)                             │
│                                                   │
│  相邻两对是同一个绕组                             │
└──────────────────────────────────────────────────┘
```

**连接步骤**:

1. 识别步进电机的两组线圈（使用万用表测量通断）
2. 将第一组线圈连接到 A+ 和 A-
3. 将第二组线圈连接到 B+ 和 B-
4. 确保方向一致（A+ 和 B+ 不能互换）

### 限位开关连接

```
限位开关类型:

1. 机械开关 (推荐入门)
   - 简单可靠
   - 需要消抖
   
2. 光学开关 (更精确)
   - 无接触磨损
   - 需要注意极性
```

**接线方法**:

```
Arduino CNC Shield 限位接口:

┌─────────────────────────────────┐
│  X-  │  X+  │  Y-  │  Y+  │  Z- │
│  (GND)                         │
└─────────────────────────────────┘

机械开关接线:
   ───┤  ├──── GND
       │      │
      COM    NC
```

**注意**: 如果使用常开 (NO) 开关，需要启用软件上拉电阻。

### 电源连接

```
⚠️ 警告: 确保电源极性正确！

Arduino CNC Shield 电源:
┌─────────────────────────────────┐
│  VMOT+  │  VMOT-  │  GND      │
│   (+)   │   (-)   │  (逻辑地) │
└─────────────────────────────────┘

电压选择:
- 12V: 适用于 NEMA 17 小型电机
- 24V: 推荐用于更大电机或更高速运行
- 绝对不要超过 36V
```

## 基本操作流程

### 1. 开机检查

```
步骤:
1. 检查所有接线是否牢固
2. 确认电源电压正确
3. 先开控制器电源，后开主轴电源
4. 观察各指示灯状态
```

### 2. 连接软件

```
1. 打开 GrblPanel 或其他控制器软件
2. 选择正确的串口 (通常为 COM3-COM5)
3. 设置波特率: 115200
4. 点击连接
5. 确认连接成功
```

### 3. 归零操作 (Homing)

归零是确定工作坐标系原点的过程：

```
操作步骤:
1. 点击 "Home" 按钮
2. 等待三轴依次移动到限位开关位置
3. 限位开关触发后，轴会缓慢退回
4. 归零完成，坐标归零

注意:
- 首次使用必须进行归零
- 归零前确保限位开关工作正常
```

### 4. 设置工件原点

工件原点是加工程序的起点：

```
方法一: 使用控制器软件
1. 手动移动刀具到工件待加工位置
2. 点击 "Set Zero" 或类似按钮
3. 选择设置的轴 (X/Y/Z)

方法二: 使用 G 代码
G10 L20 P1 X0 Y0 Z0  (设置当前点为工件原点)
```

### 5. 运行加工程序

```
步骤:
1. 加载 G 代码文件 (.nc, .gcode, .txt)
2. 检查预览是否正确
3. 确保安全距离足够
4. 点击 "Run" 开始加工
5. 监控运行状态
6. 完成后检查成品
```

## G 代码入门

### 基础运动指令

#### G0 - 快速定位

```gcode
; 快速移动到 X=50, Y=30 位置
G0 X50 Y30

; 快速移动到原点
G0 X0 Y0 Z0

; 只移动 X 轴
G0 X100
```

#### G1 - 线性进给 (切削)

```gcode
; 以 100mm/min 进给率移动到 X=50, Y=30
G1 X50 Y30 F100

; 从当前位置移动到 Z=-5
G1 Z-5 F50

; 完整的切削移动
G1 X0 Y0 Z-1 F100
```


```
1. 将轴手动移远离限位开关
2. 在软件中点击 "Unlock" 或发送 $X
3. 系统恢复正常
```

## 校准与调优

### 步进电机校准

#### 计算脉冲数

```
公式:
脉冲数/毫米 = (步进电机步数/圈) × (细分倍数) / (丝杆导程)

示例:
- 步进电机: 200 步/圈
- 驱动细分: 1/16 (MS1=1, MS2=1, MS3=1)
- 丝杆导程: 8mm/圈

计算:
脉冲数/毫米 = 200 × 16 / 8 = 400 脉冲/mm
```

#### GRBL 参数设置

```gcode
; 设置 X 轴脉冲数 (每毫米)
$100 = 400

; 设置 Y 轴脉冲数
$101 = 400

; 设置 Z 轴脉冲数 (注意方向)
$102 = 400

; 查看当前脉冲设置
$100
$101
$102
```

### 进给率校准

```
校准方法:
1. 设置目标进给率 $110 = 1000 (1000mm/min)
2. 运行测试移动 100mm
3. 测量实际移动距离
4. 计算误差百分比
5. 调整 $110 值补偿误差

公式:
新进给率 = 原进给率 × (实测距离 / 目标距离)
```

### 加减速设置

```gcode
; 最大进给率 (mm/min)
$110 = 1000

; 最大快速移动速率
$111 = 2000

; 最大 Z 轴进给率
$112 = 500

; 默认加速度 (mm/s²)
$120 = 50

; X 轴加速度
$121 = 50

; Y 轴加速度
$122 = 50

; Z 轴加速度
$123 = 20

; 最大追及速度
$130 = 200  ; X 轴行程限制
$131 = 200  ; Y 轴行程限制
$132 = 100  ; Z 轴行程限制
```

## 常见问题解决

### 1. 步进电机不转

```
可能原因:
□ 驱动模块未通电
□ 接线错误或松动
□ 驱动模块损坏
□ 电机线圈短路

解决方法:
1. 检查 VMOT 电源电压 (12-36V)
2. 检查 ENABLE 引脚状态
3. 重新接线，确保线圈连接正确
4. 测试单个电机排除故障
```

### 2. 步进电机抖动

```
可能原因:
□ 电源电压不足
□ 驱动电流设置过低
□ 脉冲频率过高
□ 电机与驱动器不匹配

解决方法:
1. 提高电源电压
2. 调整 Vref 增加驱动电流
3. 降低进给率
4. 检查电机规格匹配
```

### 3. 限位开关误触发

```
可能原因:
□ 开关类型设置错误 ($5)
□ 机械开关抖动
□ 接线过长导致干扰
□ 电压过低

解决方法:
1. 正确设置 $5 参数
2. 添加消抖电路或软件滤波
3. 使用屏蔽线
4. 检查接线
```

### 4. 串口连接失败

```
可能原因:
□ 驱动程序未安装
□ 端口被占用
□ USB 线过长或质量差
□ Arduino 固件问题

解决方法:
1. 安装正确的 USB 驱动 (CH340/FTDI)
2. 关闭其他占用串口的程序
3. 更换 USB 线
4. 重新上传固件
```

### 5. G 代码无法识别

```
可能原因:
□ G 代码格式错误
□ 控制器不支持该指令
□ 坐标超出行程范围

解决方法:
1. 检查 G 代码语法
2. 查看 GRBL 支持的指令列表
3. 确保坐标在机器行程内
4. 简化 G 代码测试
```

## 安全注意事项

### 电气安全

```
⚠️ 重要警告:

1. 电源
   - 使用稳定的直流电源
   - 电压不要超过额定值
   - 做好防短路措施

2. 接地
   - 电机驱动器可靠接地
   - 避免电磁干扰

3. 散热
   - 驱动模块安装散热片
   - 保持通风良好
```

### 机械安全

```
1. 防护措施
   - 佩戴护目镜
   - 避免手指接触运动部件
   - 固定好工件和刀具

2. 急停
   - 熟悉急停按钮位置
   - 保持紧急情况下可以快速触及

3. 环境
   - 保持工作区域整洁
   - 移除松散物品和衣物
```

### 操作安全

```
1. 启动前检查
   - 确认限位开关工作正常
   - 检查刀具安装牢固
   - 确认工件夹持稳固

2. 运行监控
   - 加工过程中保持观察
   - 注意异常声音或气味
   - 随时准备按急停

3. 停机后
   - 先关闭主轴
   - 等待运动停止
   - 关闭控制电源
```

## 硬件准备

### 必需组件

| 组件 | 规格 | 数量 | 说明 |
|------|------|------|------|
| ESP32-S3 开发板 | ESP32-S3-DevKitC-1 或兼容板 | 1 | 主控制器 |
| 步进电机 | NEMA 17 (1.8°/步) | 3 | X/Y/Z 轴 |
| 步进电机驱动 | A4988 / DRV8825 / TMC2209 | 3 | 电机驱动 |

| 电源 | 12V/24V 5A+ | 1 | 根据电机规格 |

### 可选组件

- 主轴电机 (12V/24V DC 无刷电机或电主轴)
- 冷却泵
- 急停按钮
- 电磁阀 (气动/液压)
- 探针 (工件零点校准)

### 引脚分配

```
ESP32-S3 DevKitC-1 引脚定义:

电源:
├── 5V    → 电源输入 (USB 或外部 5V)
├── 3.3V  → 内部供电
├── GND   → 地线

步进电机控制:
├── GPIO1  → X 轴步进脉冲 (Step)
├── GPIO2  → X 轴方向控制 (Dir)
├── GPIO3  → Y 轴步进脉冲 (Step)
├── GPIO4  → Y 轴方向控制 (Dir)
├── GPIO5  → Z 轴步进脉冲 (Step)
├── GPIO6  → Z 轴方向控制 (Dir)
├── GPIO7  → 电机使能控制 (Enable, 低电平使能)

限位开关:
├── GPIO8  → X 轴限位开关 (X Limit)
├── GPIO9  → Y 轴限位开关 (Y Limit)
├── GPIO10 → Z 轴限位开关 (Z Limit)

主轴控制:
├── GPIO11 → 主轴 PWM 输出 (Spindle PWM)
├── GPIO12 → 探针输入 (Probe)

其他:
├── GPIO0  → 急停输入 (E-Stop)
└── GPIO21 → 状态 LED (Status LED)
```

## WiFi 连接配置

### AP 模式 (默认)

控制器创建 WiFi 热点:

1. 使用手机/电脑搜索 WiFi 网络
2. 找到 `CNC_Controller_XXXX` (XXXX 为随机数字)
3. 输入密码: `12345678`
4. 连接成功后即可访问控制界面

### STA 模式 (连接路由器)

修改 `Config.h`:

```cpp
#define WIFI_MODE_STA
#define STA_WIFI_SSID "YourWiFiName"
#define STA_WIFI_PASSWORD "YourWiFiPassword"
```

重新烧录固件后,控制器会自动连接指定网络。

### 混合模式

同时启用 AP 和 STA:

```cpp
#define WIFI_MODE_AP_AND_STA
#define WIFI_SSID_PREFIX "CNC_Controller_"
#define WIFI_PASSWORD "12345678"
#define STA_WIFI_SSID "YourWiFiName"
#define STA_WIFI_PASSWORD "YourWiFiPassword"
```

## 故障排除

### WiFi 连接问题

#### 问题: 找不到 WiFi 热点

解决方案:
1. 检查 ESP32-S3 是否正常供电
2. 检查固件是否成功烧录
3. 观察串口输出确认 WiFi 初始化
4. 尝试重置 ESP32 (按下 EN 按钮)

#### 问题: 连接后无法访问 Web

解决方案:
1. 确认使用正确的 IP 地址
2. 检查防火墙设置
3. 尝试使用 ping 命令检测连接
4. 重启浏览器或清除缓存

### 串口通信问题

#### 问题: 串口输出乱码

解决方案:
1. 确认波特率设置为 115200
2. 检查 USB 数据线质量
3. 尝试不同的 USB 端口

#### 问题: 命令无响应

解决方案:
1. 检查是否处于报警模式 (`$X` 解锁)
2. 确认限位开关状态
3. 检查是否有错误消息

### 运动控制问题

#### 问题: 电机不转动

解决方案:
1. 检查电机驱动供电 (12V/24V)
2. 检查 ENABLE 引脚电平
3. 验证步进脉冲信号
4. 检查电机接线

#### 问题: 电机抖动

解决方案:
1. 降低步进脉冲频率
2. 调整驱动电流设置
3. 检查电源电压稳定性
4. 确认电机接线正确

#### 问题: 运动方向相反

解决方案:
修改 `Config.h` 中的方向反转设置:
```cpp
#define X_DIR_INVERT true   // X 轴方向反转
#define Y_DIR_INVERT false  // Y 轴方向保持
```

### 固件问题

#### 问题: 固件上传失败

解决方案:
1. 按住 BOOT 按钮再按 EN 进入下载模式
2. 降低上传速度试试
3. 检查 USB 驱动安装
4. 尝试使用不同的 USB 线

#### 问题: 启动循环

解决方案:
1. 检查配置文件语法
2. 减少内存使用
3. 调整分区方案

### 性能优化

#### 提高步进脉冲频率

```cpp
// Config.h
#define MAX_STEP_RATE 100000  // 100kHz
```
