## 项目简介

本项目提供两个版本的 GRBL 风格 CNC 控制器：

| 版本 | 平台 | 特点 | 适用场景 |
|------|------|------|----------|
| **Arduino 版本** | Arduino Uno/Nano/Mega | 串口通信，稳定可靠 | 入门级 CNC，桌面雕刻机 |
| **ESP32-S3 版本** | ESP32-S3 | WiFi 网络控制，性能更强 | 高级应用，无线控制需求 |

### 核心功能

- ✅ G 代码解析与执行
- ✅ 三轴步进电机控制 (X/Y/Z)
- ✅ 串口通信 (Arduino) / WiFi + 串口 (ESP32-S3)
- ✅ 实时位置显示
- ✅ 速度与进给率控制
- ✅ 原点设置与归零
- ✅ 急停功能

### Arduino 版本

cd arduino/grbl-controller

# 选择板卡: Arduino Uno/Nano

## 硬件要求

### 通用配件

| 配件 | 规格 | 数量 |
|------|------|------|
| 步进电机 | NEMA 17 或 equivalent | 3 |
| 步进电机驱动 | A4988 / DRV8825 / TB6600 | 3 |

| 电源 | 12V/24V 根据电机规格 | 1 |

### Arduino 版本额外配件

- Arduino Uno / Nano / Mega
- USB 数据线 (Type-A to Type-B 或 Type-C)

### ESP32-S3 版本额外配件

- ESP32-S3 DevKitC-1 或兼容板
- WiFi 网络环境

## G 代码支持

本控制器支持标准 G 代码指令集：

### 运动指令

| G 代码 | 功能 | 示例 |
|--------|------|------|
| G0 | 快速移动 | `G0 X10 Y20` |
| G1 | 线性进给 | `G1 X50 Y30 F100` |
| G28 | 归零 | `G28` |
| G90 | 绝对坐标 | `G90` |
| G91 | 相对坐标 | `G91` |

### 设置指令

| G 代码 | 功能 | 示例 |
|--------|------|------|
| M3 | 主轴正转 | `M3 S1000` |
| M5 | 主轴停止 | `M5` |
| F | 设置进给率 | `F200` |
| S | 设置转速 | `S5000` |

## 常见问题

### Q: 步进电机抖动或失步?

A: 检查以下设置:
- 驱动电流设置 (参考驱动模块说明书)
- 电源电压是否充足
- 步进脉冲频率是否过高
- 电机接线顺序是否正确

### Q: ESP32-S3 无法连接 WiFi?

A: 确认:
- WiFi 密码是否正确
- 路由器频段 (ESP32-S3 支持 2.4GHz)
- 信号强度

### Q: G 代码无法识别?

A: 检查:
- G 代码格式是否正确
- 是否使用了不支持的指令
- 坐标参数是否在行程范围内

### Arduino 详细接线步骤

#### 1. 电源接线

```
步骤 1: 电源正极 (+) 连接驱动板 VMOT
步骤 2: 电源负极 (-) 连接驱动板 GND
步骤 3: 在电源输入端并联 100μF 电容（注意极性）
```

**警告**: 确保电源电压与电机和驱动板规格匹配（A4988 最大 35V，DRV8825 最大 45V）

#### 2. 步进电机接线

每个轴（X/Y/Z）需要独立驱动板：

| Arduino 引脚 | A4988 引脚 | 功能 |
|-------------|-----------|------|
| D2 (X) / D3 (Y) / D4 (Z) | STEP | 脉冲输入 |
| D5 (X) / D6 (Y) / D7 (Z) | DIR | 方向控制 |
| D8 (X) / D9 (Y) / D10 (Z) | ENABLE | 使能（低电平有效） |

#### 3. 限位开关接线

| Arduino 引脚 | 限位开关 | 说明 |
|-------------|---------|------|
| D9 | X- 限位 | X 轴负向限位 |
| D10 | X+ 限位 | X 轴正向限位 |
| D11 | Y- 限位 | Y 轴负向限位 |
| D12 | Y+ 限位 | Y 轴正向限位 |
| D13 | Z- 限位 | Z 轴负向限位 |
| A0 | Z+ 限位 | Z 轴正向限位 |

#### 4. 主轴 PWM 控制接线

| Arduino 引脚 | IRF520 模块 | 说明 |
|-------------|------------|------|
| D11 | PWM 输入 | 主轴转速控制 |
| VIN | VCC (5V) | 驱动板电源 |
| GND | GND | 共地 |

## 接线注意事项

### 安全规范

1. **电源安全**
   - 接线前确保电源已关闭
   - 确认输入电压与设备规格匹配
   - 在电源输入端安装保险丝（建议 5A）
   - 使用合适的线径（建议 18AWG 以上用于电源线）

2. **防静电措施**
   - 操作前释放身体静电
   - 避免触摸集成电路引脚
   - 使用防静电手环（推荐）

3. **散热管理**
   - 确保驱动芯片安装散热片
   - 保持良好通风环境
   - 温度超过 60°C 时考虑增加风扇

### 接线质量

1. **连接可靠性**
   - 使用螺丝端子确保牢固连接
   - 杜邦线仅用于临时测试
   - 焊锡连接点确保光滑无虚焊

2. **信号完整性**
   - 电机驱动信号线尽量短
   - 避免信号线与电源线平行走线
   - 敏感信号使用屏蔽线

3. **标识建议**
   - 每个接线点使用标签标识
   - 绘制接线图存档
   - 保留接线照片

### 接地处理

```
正确的接地布局:

主电源负极 ──────────┬──→ 驱动板 GND
                      │
                      ├──→ ESP32/Arduino GND
                      │
                      ├──→ 限位开关 GND
                      │
                      └──→ 主轴模块 GND
```

## 常见问题排查

### 问题 1: 电机不转动

**检查步骤:**
1. 测量驱动板输入电压是否正常（12V/24V）
2. 检查 STEP 信号是否有脉冲输出（用万用表或示波器）
3. 确认 ENABLE 引脚电平正确（A4988 低电平使能）
4. 检查电机线圈是否开路（测量电机接线端子电阻）
5. 检查驱动板是否过热保护

### 问题 2: 电机抖动但不转动

**可能原因:**
1. 脉冲频率过高，尝试降低速度
2. 驱动电流设置过低（调节驱动板电位器）
3. 电机线圈接线错误（检查相序）
4. 电源功率不足

### 问题 3: 限位开关不工作

**排查方法:**
1. 检查限位开关类型（常开/常闭）
2. 测量限位开关输出信号
3. 检查 GRBL/$5 设置（限位开关电平）
4. 检查接线是否松动

### 问题 4: ESP32 WiFi 连接不稳定

**解决方案:**
1. 确保 WiFi 天线正确连接
2. 减少与路由器之间的距离
3. 避免周围强干扰源
4. 检查供电稳定性（WiFi 模块电流峰值可达 500mA）

### 问题 5: 主轴转速异常

**检查项目:**
1. PWM 信号频率是否正确（通常 1-30kHz）
2. 检查主轴控制模块是否损坏
3. 测量 PWM 占空比与电压对应关系
4. 确认主轴电机规格与控制器匹配

## 硬件准备

- Arduino Uno/Nano (ATmega328P)

### 详细说明

#### G0 - 快速定位
```
G0 X10 Y10 Z5
```
快速移动到指定位置，不进行切削。运动速度由系统最大速度决定。

#### G1 - 线性进给
```
G1 X50 Y30 Z-2 F200
```
以指定进给速度（F200 mm/min）移动到目标位置，用于切削加工。

#### G2/G3 - 圆弧运动
```
G2 X30 Y20 I10 J0 F100  ; 顺时针圆弧到(30,20)，圆心相对起点偏移(10,0)
G3 X30 Y20 I10 J0 F100  ; 逆时针圆弧到(30,20)
```
- G2: 顺时针圆弧
- G3: 逆时针圆弧
- I: 圆心相对于起点的X偏移
- J: 圆心相对于起点的Y偏移
- K: 圆心相对于起点的Z偏移（用于螺旋）

#### G4 - 延时
```
G4 P500  ; 延时500毫秒
```

#### G28/G30 - 返回参考点
```
G28      ; 直接返回参考点
G28 X10 Y10  ; 经过中间点(10,10)后返回参考点
G30      ; 返回第二参考点
```

#### G38.x - 探针功能
```
G38.2 Z-10 F50  ; 向Z负方向探查，碰到传感器后停止
```
用于工件找零、对齐和高度测量。

## G代码格式说明

### 基本格式
```
N001 G1 X10.5 Y20.0 F200
```

- **N**: 行号（可选）
- **G1**: G代码命令
- **X10.5 Y20.0**: 坐标参数
- **F200**: 进给速度

### 模态组

GRBL使用模态组来组织命令，同一组的命令会互相替代：

| 模态组 | 包含命令 |
|--------|----------|
| 运动 | G0, G1, G2, G3, G38.2, G38.3, G38.4, G38.5, G80 |
| 坐标系 | G54, G55, G56, G57, G58, G59, G59.1, G59.2, G59.3 |
| 平面选择 | G17, G18, G19 |
| 距离模式 | G90, G91 |
| 弧距离模式 | G90.1, G91.1 |
| 速度模式 | G93, G94 |
| 单位 | G20, G21 |
| 刀具补偿 | G40 |
| 行程补偿 | G98, G99 |
| 主轴 | M3, M4, M5 |
| 冷却液 | M7, M8, M9 |
| 覆盖 | M48, M49 |

# ESP32-S3 WiFi 控制功能说明

本文档详细说明 ESP32-S3 版本 GRBL CNC 控制器的 WiFi 网络控制功能。

## 功能概述

ESP32-S3 版本支持通过 WiFi 网络进行远程控制，支持以下通信方式：

- **WebSocket** - 实时双向通信，适合控制界面
- **HTTP REST API** - 标准化 API 接口
- **Telnet** - 兼容传统串口协议

## WiFi 连接配置

### 方式一：使用配置文件

编辑 `esp32_s3/wifi_config.h`：

```cpp
// WiFi 配置
#define WIFI_SSID "你的WiFi名称"
#define WIFI_PASSWORD "你的WiFi密码"

// 静态 IP 配置（可选）
#define USE_STATIC_IP true
#define STATIC_IP 192, 168, 1, 100
#define GATEWAY 192, 168, 1, 1
#define SUBNET 255, 255, 255, 0
#define DNS 8, 8, 8, 8
```

### 方式二：通过 Web 配网

首次运行时，ESP32-S3 会创建热点：

1. 搜索 WiFi 网络 `GRBL-CNC-XXXX`
2. 连接后打开浏览器访问 `192.168.4.1`
3. 在配置页面输入 WiFi 凭据
4. 设备将自动重启并连接

### 方式三：AP 模式（热点模式）

```cpp
// 在 wifi_config.h 中启用 AP 模式
#define USE_AP_MODE true
#define AP_SSID "GRBL-CNC-Controller"
#define AP_PASSWORD "12345678"
```

设备将创建独立热点，可直接连接控制。

## 通信协议

### WebSocket

默认端口：**81**

连接地址：`ws://<ESP32_IP>:81`

#### 发送命令格式

```json
{
  "type": "command",
  "cmd": "G0 X10 Y10",
  "id": 1
}
```

#### 接收数据格式

```json
{
  "type": "response",
  "data": "ok",
  "id": 1
}
```

#### 状态更新

```json
{
  "type": "status",
  "data": {
    "state": "Idle",
    "pos": {"x": 0, "y": 0, "z": 0},
    "wco": {"x": 0, "y": 0, "z": 0},
    "buf": 15,
    "rx": 0
  }
}
```

### HTTP REST API

默认端口：**80**

#### 端点列表

| 方法 | 端点 | 描述 |
|------|------|------|
| GET | `/api/status` | 获取控制器状态 |
| GET | `/api/position` | 获取当前位置 |
| POST | `/api/command` | 发送 G-code 命令 |
| GET | `/api/settings` | 获取 GRBL 设置 |
| POST | `/api/settings` | 修改设置 |
| GET | `/api/version` | 获取固件版本 |

#### 示例请求

**发送 G-code 命令：**

```bash
curl -X POST http://192.168.1.100/api/command \
  -H "Content-Type: application/json" \
  -d '{"cmd": "G0 X50 Y50"}'
```

**响应：**
```json
{
  "status": "ok",
  "response": "ok"
}
```

**获取状态：**

```bash
curl http://192.168.1.100/api/status
```

**响应：**
```json
{
  "state": "Run",
  "position": {"x": 25.5, "y": 10.0, "z": 0.0},
  "buffer": 12,
  "feed": 1000,
  "spindle": 0
}
```

### Telnet 协议

默认端口：**23**

使用兼容传统串口终端连接：

```bash
telnet 192.168.1.100 23
```

## Web 控制界面

设备内置 Web 服务器，可通过浏览器访问：

1. 打开浏览器
2. 输入 ESP32-S3 的 IP 地址
3. 进入 Web 控制界面

### 界面功能

- **实时状态显示** - 坐标、状态、进度
- **G-code 发送** - 命令输入和文件上传
- **手轮控制** - 手动移动轴
- **参数配置** - GRBL 参数设置
- **文件管理** - 上传/删除 G-code 文件

## 使用示例

### Python 控制示例

```python
import websocket
import json
import time

def on_message(ws, message):
    data = json.loads(message)
    print(f"收到: {data}")

def on_open(ws):
    # 发送命令
    ws.send(json.dumps({"type": "command", "cmd": "?", "id": 1}))
    time.sleep(0.5)
    ws.send(json.dumps({"type": "command", "cmd": "G0 X0 Y0", "id": 2}))
    time.sleep(0.5)
    ws.send(json.dumps({"type": "command", "cmd": "G1 Z-5 F100", "id": 3}))

# WebSocket 连接
ws = websocket.WebSocketApp(
    "ws://192.168.1.100:81",
    on_message=on_message,
    on_open=on_open
)
ws.run_forever()
```

### Node-RED 集成

在 Node-RED 中使用 HTTP 请求节点：

1. 添加 HTTP Request 节点
2. 配置 URL：`http://192.168.1.100/api/command`
3. Method：POST
4. Body：`{"cmd": "G0 X10"}`

## 故障排除

### 无法连接 WiFi

1. 检查 SSID 和密码是否正确
2. 确认 WiFi 信号强度
3. 检查路由器是否启用 2.4GHz（ESP32 不支持 5GHz）

### WebSocket 连接失败

1. 确认防火墙允许 WebSocket 端口
2. 检查 ESP32 IP 地址是否正确
3. 尝试刷新页面重新连接

### 延迟过高

1. 检查网络质量
2. 减少同时连接的客户端数量
3. 考虑使用有线网络连接

## 安全建议

### 网络隔离

- 建议将 CNC 控制器放在独立 VLAN
- 避免直接暴露在公网

### 访问控制

```cpp
// 启用连接密码验证

```

### HTTPS 配置
